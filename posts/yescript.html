🔑 Purpose: It is used to securely hash passwords before storing them in /etc/shadow (on Linux) or other authentication systems.

🧠 Memory-hard: Like scrypt, it is designed to be resistant to brute-force attacks using GPUs, FPGAs, or ASICs by requiring a significant amount of memory and CPU time to compute.

🔒 Improvements over scrypt:

  • Supports additional security features like ROM port hardening.
  • Offers tunable parameters to make attacks harder.
  • Better defense against tradeoff attacks (where an attacker reduces memory use at the cost of more CPU).
